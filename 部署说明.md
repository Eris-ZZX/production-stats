# 生产数据统计系统（制程版）— 部署说明

## 一、交付物

```
production-stats-deploy.zip
```

解压后目录结构：

```
production-stats/
├── package.json           # 前端依赖
├── index.html             # 入口 HTML
├── vite.config.ts         # 构建配置
├── tsconfig.json          # TypeScript 配置
├── tsconfig.app.json
├── tsconfig.node.json
├── eslint.config.js
├── public/
│   └── favicon.svg
├── src/                   # 前端源码
├── server/
│   ├── package.json       # 后端依赖
│   ├── tsconfig.json
│   └── src/               # 后端源码
│       ├── index.ts       # 服务入口
│       ├── db.ts          # 数据库初始化
│       ├── auth.ts        # JWT 认证

```

## 二、环境要求

| 项目 | 要求 |
|------|------|
| Node.js | ≥ 18.x |
| npm | 随 Node.js 自带 |
| OS | Windows / Linux |
| 磁盘 | 约 200 MB（含 node_modules） |
| 内存 | ≥ 512 MB |

### ⚠️ 重要前置条件

`better-sqlite3` 是 C++ 原生模块，安装时需要编译工具：

**Windows：**
- [Visual Studio Build Tools](https://visualstudio.microsoft.com/downloads/#build-tools-for-visual-studio-2022)（勾选"Desktop development with C++"）
- Python 3.x（安装时勾选"Add to PATH"）
- 或直接用管理员终端运行：`npm install --global windows-build-tools`

**Linux（Ubuntu/Debian）：**
```bash
sudo apt update
sudo apt install build-essential python3
```

## 三、首次部署

### Step 1 — 解压

```bash
unzip production-stats-deploy.zip -d production-stats
cd production-stats
```

### Step 2 — 安装依赖

```bash
# 1. 前端依赖
npm install

# 2. 后端依赖
cd server
npm install
cd ..
```

### Step 3 — 构建前端

```bash
npm run build
```

> 构建产物在 `dist/` 目录，后端会直接托管它。

### Step 4 — **设置密钥（必须！）**

```bash
# Linux / Mac
export JWT_SECRET="你的随机字符串至少32位"
# 示例：export JWT_SECRET="a7Kx2Mp9Qw4Rt6Yv1Bn3Cz8Lg5Df0HjE"

# Windows CMD
set JWT_SECRET=你的随机字符串至少32位
```

> **不设置将使用不安全默认值，任何人都可以伪造登录令牌。**

### Step 5 — 启动

```bash
cd server
npx tsx src/index.ts
```

启动后会看到：

```
Server running on http://localhost:3001
Frontend served from: D:\xxx\production-stats\dist
```

浏览器打开 `http://服务器IP:3001` 即可访问。

## 四、后台运行

### Windows — NSSM 服务（推荐）

1. 下载 [NSSM](https://nssm.cc/download)
2. 以管理员身份打开 CMD：

```bat
nssm install ProductionStats

# Application Path:  C:\Program Files\nodejs\node.exe (where node 查看)
# Startup Directory: D:\生产数据表\production-stats\server
# Arguments:         D:\生产数据表\production-stats\server\node_modules\tsx\dist\cli.mjs src/index.ts
# Environment:       JWT_SECRET=你的密钥; PORT=3001
```

3. `nssm start ProductionStats`

### Linux — systemd 服务（推荐）

创建 `/etc/systemd/system/production-stats.service`：

```ini
[Unit]
Description=Production Stats System
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/production-stats/server
Environment=JWT_SECRET=你的密钥
Environment=PORT=3001
ExecStart=/usr/bin/npx tsx src/index.ts
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now production-stats
sudo systemctl status production-stats
```

### Linux — PM2（轻量方案）

```bash
npm install -g pm2
cd /opt/production-stats/server
JWT_SECRET=你的密钥 pm2 start src/index.ts --interpreter npx --interpreter-args "tsx" --name production-stats
pm2 save
pm2 startup
```

## 五、端口与防火墙

默认端口：**3001**

修改端口：
```bash
# Linux / Mac
export PORT=8080

# Windows
set PORT=8080
```

防火墙放行：
```bash
# Linux (firewalld)
sudo firewall-cmd --add-port=3001/tcp --permanent
sudo firewall-cmd --reload

# Linux (iptables)
sudo iptables -A INPUT -p tcp --dport 3001 -j ACCEPT

# Windows
netsh advfirewall firewall add rule name="ProductionStats" dir=in action=allow protocol=TCP localport=3001
```

## 七、数据库

- 数据库文件：`server/data/app.db`（SQLite）
- 首次启动自动创建（含种子数据）
- **无数据库服务进程**，是一个普通文件

### 备份

```bash
cp server/data/app.db server/data/app.db.bak.$(date +%Y%m%d)
```

### 恢复

```bash
cp server/data/app.db.bak.20260609 server/data/app.db
```

## 八、更新流程

1. 停止服务
2. 覆盖更新 `src/` 和 `server/src/` 目录
3. `npm run build`（前端有变更时）
4. 启动服务

> 数据库文件 `app.db` 不会被覆盖，历史数据不丢失。

## 九、常见问题

### Q: better-sqlite3 安装报错

检查 C++ 编译工具是否安装（见第二章）。临时降级方案：
```bash
cd server
npm install better-sqlite3@9.6.0 --build-from-source
```

### Q: Windows 上 tsx 找不到

```bash
npx tsx src/index.ts     # 用 npx 前缀
# 或手动指定路径
node node_modules/tsx/dist/cli.mjs src/index.ts
```

### Q: 页面访问 404

检查 `dist/` 目录是否存在，确认已执行 `npm run build`。

### Q: 登录报错 / token 无效

检查 `JWT_SECRET` 环境变量是否在重启后仍然生效。如果用 NSSM/systemd 托管，确认配置文件中设置了环境变量。
